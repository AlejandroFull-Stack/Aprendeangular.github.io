1. Introducción

Angular es un framework de JavaScript (concretamente de TypeScript) que fue creado y es mantenido por Google. A lo largo de los años, se ha convertido en una de las herramientas más populares para el desarrollo de aplicaciones web de tipo Single Page Applications (SPA).

La principal ventaja de Angular radica en su arquitectura basada en componentes, que promueve la reusabilidad y la modularidad del código. Al usar TypeScript, obtendrás además tipado estático y un conjunto de características que te facilitan la escritura de código mantenible, especialmente para proyectos de gran envergadura.

En esta guía para principiantes, cubriremos la mayoría de los pasos básicos para que puedas, desde cero, crear tu primera aplicación, comprender la estructura de archivos de un proyecto Angular, y llegar hasta el despliegue en un entorno de producción. A medida que avances, descubrirás cómo Angular te permite:

  • Dividir la aplicación en módulos y componentes independientes.
  • Enlazar datos de forma declarativa (data-binding).
  • Utilizar servicios y hacer peticiones HTTP a APIs externas.
  • Navegar entre secciones con el sistema de rutas (routing).
  • Empaquetar y desplegar tu aplicación fácilmente.

2. Requisitos Previos

Aunque Angular se puede aprender desde cero, es muy recomendable que cuentes con algunos conocimientos básicos antes de embarcarte en el desarrollo de aplicaciones complejas. Por ejemplo:

  • Fundamentos de JavaScript/TypeScript: Angular hace uso intensivo de TypeScript, así que es importante entender variables, funciones, clases, interfaces y algunos conceptos de programación orientada a objetos.
  • HTML y CSS: Aunque Angular facilita la organización de vistas y estilos, necesitas tener claro cómo funciona la maquetación web básica y los selectores CSS.
  • Manejo de la Terminal/CLI: Angular CLI es una herramienta poderosa que te permite crear, compilar y desplegar proyectos Angular de manera rápida. Familiarizarte con la terminal te hará el proceso mucho más fluido.

Si ya cumples con estos requisitos, genial. Si no, no te preocupes: puedes ir aprendiendo a la par que exploras Angular. Lo importante es tener la motivación para seguir investigando y construyendo proyectos.

3. Instalación

3.1 Instalando Node.js y Angular CLI

Para comenzar, necesitas tener Node.js instalado en tu sistema, preferiblemente en su versión LTS (por ejemplo, Node 14 o superior). Node incluye npm, el gestor de paquetes que nos permitirá instalar la herramienta de línea de comandos de Angular (@angular/cli). 


# Instala Angular CLI globalmente en tu sistema:
npm install -g @angular/cli

# Verifica la versión instalada:
ng version

Con estos pasos, tendrás disponible el comando ng para generar nuevos proyectos, componentes y mucho más.

3.2 Creando tu primer proyecto

Después de instalar el CLI, puedes crear un nuevo proyecto de Angular con: 


ng new mi-primera-app

Este comando te solicitará que elijas algunas configuraciones como el estilo que deseas usar (CSS, SCSS, etc.) y si quieres agregar routing. Para este ejemplo, podemos decir que “sí” a todo, o ajustarlo a nuestras preferencias. Una vez completado el proceso, tendrás una estructura de directorios lista para empezar a trabajar.

4. Hola Mundo

El clásico “Hola Mundo” es la forma más sencilla de confirmar que tu entorno de desarrollo está funcionando correctamente. Para ello, ejecuta: 


cd mi-primera-app
ng serve --open

Este comando lanzará un servidor local y abrirá automáticamente la aplicación en tu navegador en la dirección http://localhost:4200. Deberías ver la página de bienvenida de Angular, donde verás la versión de Angular y un montón de enlaces y recursos.

Para personalizar tu “Hola Mundo”, edita el archivo src/app/app.component.html y borra todo su contenido para colocar algo así: 


        <h1>¡Hola Mundo desde Angular!</h1>
        <p>Este es mi primer proyecto.</p>

Guarda el archivo y verás que el navegador se recarga automáticamente, mostrando ahora tu propio texto. ¡Felicidades! Has creado tu primer Hola Mundo en Angular.

5. Estructura del Proyecto

Cuando creas una aplicación con Angular CLI, obtendrás un conjunto de carpetas y archivos que constituyen la base de tu proyecto. Es importante entender para qué sirve cada parte:

  • src/: Directorio principal con el código de la aplicación.
    • app/: Contiene los componentes, servicios, módulos principales de tu aplicación.
    • assets/: Para imágenes, fuentes, íconos o cualquier otro archivo estático.
    • environments/: Para definir variables según entorno (desarrollo, producción, etc.).
  • angular.json: Archivo de configuración del CLI, donde defines estilos, scripts y otras opciones de compilación.
  • package.json: Contiene metadatos del proyecto y las dependencias instaladas.
  • tsconfig.json: Configuración de TypeScript para tu proyecto.

Para proyectos grandes, normalmente dividirás tu aplicación en múltiples módulos para organizar el código de forma más mantenible, por ejemplo: Módulo de Autenticación, Módulo de Usuarios, Módulo de Productos, etc.

6. Creación de Componentes

6.1 ¿Qué es un Componente?

Un componente en Angular es una pieza de la interfaz de usuario (UI) y de la lógica que la acompaña. Cada componente consta de:

  • Un archivo .ts que contiene la clase y la lógica del componente.
  • Una plantilla .html que describe la vista (UI).
  • Un archivo .css o .scss que contiene los estilos específicos.

Gracias a la inyección de dependencias, los componentes también pueden comunicarse fácilmente con servicios para obtener datos, ejecutar lógica de negocio, etc.

6.2 Generando un Nuevo Componente

Para crear un nuevo componente, utiliza el CLI con el siguiente comando: 


ng generate component mi-componente

Esto generará cuatro archivos en una carpeta llamada mi-componente dentro de app/:

  • mi-componente.component.ts
  • mi-componente.component.html
  • mi-componente.component.css
  • mi-componente.component.spec.ts (para pruebas unitarias)

El archivo .spec.ts es opcional para comenzar, pero es muy útil a largo plazo para mantener pruebas unitarias de cada componente.

6.3 Ejemplo de Lógica y Plantilla

Supongamos que queremos que nuestro componente muestre un mensaje de bienvenida con un contador que aumente cada vez que hagamos clic en un botón. Podríamos editar mi-componente.component.ts así: 


              // mi-componente.component.ts

              import { Component } from '@angular/core';

              @Component({
                selector: 'app-mi-componente',
                templateUrl: './mi-componente.component.html',
                styleUrls: ['./mi-componente.component.css']
              })
              export class MiComponenteComponent {
                mensaje: string = '¡Bienvenido a Mi Componente!';
                contador: number = 0;

                incrementarContador() {
                  this.contador++;
                }
              }

Luego, en la plantilla (mi-componente.component.html): 


          <h2>{{ mensaje }}!</h2>
          <p>Has hecho clic {{ contador }} veces.</p>

Nota cómo usamos {{ }} para mostrar las variables mensaje y contador, y cómo detectamos el evento (click) para llamar al método incrementarContador(). Este es el data-binding y la interacción de eventos que hace tan poderoso a Angular.

7. Uso de Servicios y Peticiones HTTP

En una aplicación real, rara vez tenemos todos los datos en el propio cliente. Normalmente, debemos comunicarnos con APIs externas o con nuestro servidor para obtener datos (usuarios, productos, configuraciones, etc.). Para ello, Angular nos provee la clase HttpClient, que se utiliza inyectándola en un servicio.

7.1 Creando un Servicio

Usamos el CLI nuevamente: 


ng generate service servicios/mi-servicio

Esto creará un archivo mi-servicio.service.ts y un archivo de pruebas mi-servicio.service.spec.ts. Dentro de mi-servicio.service.ts, podrías tener algo así: 


// mi-servicio.service.ts

import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class MiServicioService {

  private baseUrl = 'https://api.miapp.com'; // URL de tu API

  constructor(private http: HttpClient) { }

  obtenerDatos(): Observable {
    return this.http.get(`${this.baseUrl}/datos`);
  }

  enviarDatos(payload: any): Observable {
    return this.http.post(`${this.baseUrl}/nuevo-dato`, payload);
  }

}

Este servicio define métodos para GET y POST requests. Recuerda importar HttpClientModule en tu app.module.ts para poder usar HttpClient.

7.2 Consumiendo el Servicio en un Componente

Una vez creado el servicio, podemos inyectarlo en cualquier componente. Por ejemplo: 


// mi-componente.component.ts (continuando ejemplo anterior)

import { Component, OnInit } from '@angular/core';
import { MiServicioService } from '../servicios/mi-servicio.service';

@Component({
  selector: 'app-mi-componente',
  templateUrl: './mi-componente.component.html',
  styleUrls: ['./mi-componente.component.css']
})
export class MiComponenteComponent implements OnInit {
  datos: any[] = [];
  errorMensaje: string = '';

  constructor(private miServicio: MiServicioService) {}

  ngOnInit(): void {
    this.miServicio.obtenerDatos().subscribe({
      next: (resultado) => {
        this.datos = resultado;
      },
      error: (error) => {
        this.errorMensaje = 'Error al obtener datos: ' + error.message;
      }
    });
  }
}

Así, cuando el componente se inicie (ngOnInit()), se hará la petición HTTP a nuestra API, y almacenaremos el resultado en this.datos. En la plantilla (mi-componente.component.html), podrías mostrar esos datos, por ejemplo: 


          <div *ngIf="errorMensaje">
          <p>{{ errorMensaje }}</p>
          </div>

          <ul>
          <li *ngFor="let item of datos">
    {{ item.nombre }}
    </li>
    </ul>

8. Ruteo en Angular

Uno de los pilares de las SPA (Single Page Applications) es la habilidad de navegar entre diferentes vistas sin recargar por completo la página. Angular provee el RouterModule, que te permite definir rutas y asociarlas con componentes específicos.

Para usar el ruteo, generalmente en tu app.module.ts, creas un arreglo de rutas y lo importas: 


            import { NgModule } from '@angular/core';
            import { BrowserModule } from '@angular/platform-browser';
            import { RouterModule, Routes } from '@angular/router';
            import { AppComponent } from './app.component';
            import { MiComponenteComponent } from './mi-componente/mi-componente.component';
            import { OtroComponenteComponent } from './otro-componente/otro-componente.component';

            const routes: Routes = [
              { path: '', component: MiComponenteComponent },
              { path: 'otro', component: OtroComponenteComponent }
            ];

            @NgModule({
              declarations: [
                AppComponent,
                MiComponenteComponent,
                OtroComponenteComponent
              ],
              imports: [
                BrowserModule,
                RouterModule.forRoot(routes)  // Importa el ruteo aquí
              ],
              providers: [],
              bootstrap: [AppComponent]
            })
            export class AppModule { }

En tu plantilla principal (app.component.html), usas <router-outlet> para indicar dónde se cargará el contenido de cada ruta: 


        <nav>
        <a routerLink="/">Inicio</a>
        <a routerLink="/otro">Página Alternativa</a>
        </nav>


<router-outlet></router-outlet>

Con esto, cuando el usuario navegue a /, se cargará MiComponenteComponent, y cuando vaya a /otro, se cargará OtroComponenteComponent. ¡Sin recargar por completo la aplicación!

9. Despliegue

Una vez que tu aplicación está lista para producción, Angular CLI ofrece un comando de build optimizado: 


ng build --prod

Esto generará una carpeta dist/ con los archivos minificados y optimizados de tu aplicación. Podrás llevar esa carpeta a un servidor web (por ejemplo, Apache, NGINX, GitHub Pages, Firebase Hosting, etc.) sin complicaciones. Sólo recuerda que si usas un sistema de ruteo con RouterModule, puede que necesites una configuración especial de reescritura de URLs para que tu aplicación maneje correctamente las rutas en el lado del cliente.

Hoy en día existen múltiples opciones para desplegar aplicaciones Angular de manera sencilla:

  • Firebase Hosting: Gratis hasta cierto límite, muy integrado con las herramientas de Google.
  • GitHub Pages: Ideal para proyectos de demostración o documentación.
  • Netlify y Vercel: Servicios de despliegue continuo muy populares con planes gratuitos.
  • Servidores propios o VPS: Requiere mayor configuración manual, pero te brinda total control.

10. Conclusiones

A lo largo de estas secciones, hemos recorrido los principales aspectos de Angular: desde la instalación, el clásico Hola Mundo, la estructura de archivos, la creación de componentes y servicios, hasta el sistema de ruteo y la publicación de una aplicación en producción.

Lo más valioso de Angular es su ecosistema y la filosofía de agrupar todo lo necesario para construir una aplicación moderna (inyección de dependencias, manejo de rutas, comunicación HTTP, compilación optimizada, etc.) de forma unificada y coherente.

Si has llegado hasta aquí, ya tienes las bases para embarcarte en proyectos más ambiciosos y explorar temas avanzados como:

  • Lazy loading de módulos: para optimizar el rendimiento y cargar sólo lo necesario.
  • State Management con NgRx o Akita: manejar el estado de la aplicación de forma predecible y escalable.
  • Testing con Jasmine y Karma: asegurar la calidad del código con pruebas unitarias y de integración.
  • Internacionalización (i18n): para crear aplicaciones multi-idioma.

Espero que esta guía te haya sido de utilidad y te animes a seguir profundizando en el maravilloso mundo de Angular. Recuerda que la mejor forma de aprender es experimentar, así que no dudes en crear tus propios proyectos, probar librerías de terceros y sumergirte en la comunidad de desarrolladores que cada día colabora para que Angular siga evolucionando.

¡Feliz aprendizaje y mucha suerte en tus futuros desarrollos!
